# Security Policy

## Supported Versions

The following versions of this project are currently supported with security updates:

| Version | Supported |
|--------|-----------|
| Latest | ✅        |
| Older | ❌        |

Only the latest released version receives security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, report it responsibly by following these steps:

1. Send an email describing the vulnerability in detail.
2. Include:
   - Steps to reproduce
   - Potential impact
   - Any relevant logs or screenshots
3. Allow time for the issue to be investigated and fixed before public disclosure.

## Disclosure Process

Once a security report is received:
1. The report will be acknowledged within a reasonable timeframe.
2. The issue will be investigated and validated.
3. A fix will be developed and tested.
4. A security release will be published if necessary.
5. Credit will be given to the reporter (if desired).

## Thanks

We appreciate responsible disclosure and the efforts of the security community to help keep this project safe.
